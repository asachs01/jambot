Chord Chart Formats

JamBot generates PDF chord charts in a specific format designed for quick readability during jam sessions. This page explains the format structure and how to read the charts.

TNBGJ Songbook Format

The default format is based on the Thursday Night Bluegrass Jam (TNBGJ) songbook style, optimized for bluegrass, folk, and old-time music. It's designed to be read at a glance during live jams.

Key Characteristics

  • Landscape orientation – Fits more chord columns on a single page
  • Two-panel layout – Lyrics on the left, chord grid on the right
  • Column-major reading – Read chords top-to-bottom, then left-to-right
  • Multiple keys – Shows the same song transposed to common jam keys
  • Serif typography – Georgia/Times fonts for traditional songbook feel

Reading the Chord Grid

The chord grid uses column-major order, which is different from reading a book. Here's how to read it:

Reading Order: ┌─────┬─────┬─────┬─────┐ │ 1 │ 5 │ 9 │ 13 │ Read DOWN each column, │ 2 │ 6 │ 10 │ 14 │ then move RIGHT to the │ 3 │ 7 │ 11 │ 15 │ next column. │ 4 │ 8 │ 12 │ 16 │ └─────┴─────┴─────┴─────┘ Example (32 chord progression): ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐ │ G │ G │ G │ G │ C │ C │ C │ C │ │ G │ G │ G │ G │ C │ C │ C │ C │ │ G │ G │ G │ G │ C │ C │ C │ C │ │ G │ G │ D │ D │ G │ G │ G │ G │ └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘ Plays as: G G G G | G G G D | G G G D | G G D D | C C C G | ...
Tip: For bluegrass jams, each cell typically represents one beat (4 cells = 1 measure in 4/4 time). The grid naturally groups into 4-bar and 8-bar phrases.

Section Labels

Sections are labeled above their chord grids:

  • Verse – Sung section
  • Chorus – Repeated sung section
  • Verse/Chorus – Same chords for both (common in bluegrass)
  • Break – Instrumental solo section
  • Intro – Opening chords before vocals
  • Tag – Ending phrase (often repeated)

Ending Markers

For songs with first/second endings, circled numbers appear above the columns indicating which ending to play:

① ② ┌─────┬─────┬─────┬─────┐ │ G │ C │ D │ G │ ① = First time through │ G │ C │ G │ G │ ② = Second time (ending) └─────┴─────┴─────┴─────┘

Multiple Keys

Charts automatically display the same song in multiple keys. This is essential for jam sessions where different instruments or vocalists prefer different keys.

Common Key Pairings for Bluegrass

  • G and A – Most common; banjo-friendly keys
  • C and D – Common for vocal-forward songs
  • D and E – Higher energy keys
  • A and B – For songs that need extra brightness

The transposition is handled automatically by JamBot's chord engine, respecting sharp/flat conventions for each key signature.

Draft vs. Approved Charts

Charts go through a review workflow:

Status Appearance Meaning
draft "DRAFT" watermark at bottom AI-generated, needs human review
approved No watermark Verified by jam leader

Jam leaders can approve draft charts using the /jambot-chart approve command.

Technical Details

For developers and curious users, here are the technical specs:

PDF Specifications

  • Page size: Letter (11" × 8.5") landscape
  • Margins: 0.5" all sides
  • Panel gap: 0.4"
  • Grid cell size: 28pt × 24pt
  • Title font: Georgia Bold 18pt (lyrics panel)
  • Chart title font: Georgia Bold Italic 22pt
  • Chord font: Georgia Bold 11pt
  • Lyrics font: Georgia Regular 11pt

Data Model

Charts are stored internally as JSON:

{
  "title": "Blue Ridge Cabin Home",
  "chart_title": "Blue Ridge Cabin Home",
  "keys": [
    {
      "key": "G",
      "sections": [
        {
          "label": "Verse/Chorus",
          "rows": 4,
          "endings": null,
          "chords": ["G", "G", "G", "G", "G", "G", "D", "D", ...]
        }
      ]
    },
    {
      "key": "A",
      "sections": [...]
    }
  ],
  "lyrics": [
    {
      "label": "Verse 1",
      "lines": [
        "There's a well beaten path in the old mountainside",
        "Where I wandered when I was a lad",
        ...
      ]
    }
  ],
  "status": "approved"
}

This format allows for programmatic manipulation, transposition, and rendering to different output formats.
