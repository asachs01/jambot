Configuration Guide

JamBot supports two configuration methods: modal-based configuration through Discord (recommended) and legacy environment variables.

RecommendedModal-Based Configuration

Configure JamBot directly through Discord slash commands - no server restart required!

Step 1: Create Your Spotify Developer App

‚ö†Ô∏è Important: Each Discord server needs its own Spotify Developer app to avoid rate limiting and potential account bans. Do NOT share Spotify credentials across multiple servers.
  1. Go to Spotify Developer Dashboard
  2. Click "Create App"
  3. Fill in App Name (e.g., "MyServer JamBot") and Description
  4. Add Redirect URI: https://your-jambot-domain.com/callback
  5. Save and copy your Client ID and Client Secret

Step 2: Get User IDs

You'll need Discord user IDs for your jam leaders and approvers. Use the /jambot-getid command:

/jambot-getid @username

This will display the user's unique Discord ID. Copy this ID for the next step.

Step 3: Configure JamBot

Server administrators can open the configuration modal with the /jambot-setup command:

/jambot-setup

This opens a modal with the following essential fields:

  • Jam Leader User IDs - Users who can post setlists (comma-separated)
  • Song Approver User IDs - Users who can approve songs (comma-separated)
  • Spotify Client ID - From your Spotify Developer Dashboard (Step 1)
  • Spotify Client Secret - From your Spotify Developer Dashboard (stored securely)
  • Spotify Redirect URI (optional) - Leave blank to use default web server URL
üí° Advanced Settings: For optional playlist settings (playlist channel, name template), use /jambot-settings after completing basic setup.

Step 4: Authorize Spotify

Finally, authorize JamBot to create playlists on your behalf with /jambot-spotify-setup:

/jambot-spotify-setup

This will send you a private authorization link. Click it to connect your Spotify account.

Example Configuration:
Jam Leaders: 123456789012345678, 987654321098765432
Approvers: 111222333444555666, 777888999000111222

Features

  • ‚úÖ Multiple Users - Support for multiple jam leaders and approvers
  • ‚úÖ Real-Time Validation - IDs are validated immediately
  • ‚úÖ Per-Server Storage - Configuration stored in database per Discord server
  • ‚úÖ No Restart Required - Changes take effect immediately
  • ‚úÖ Admin-Only - Only administrators can modify configuration

Permissions

Only users with Administrator permissions in the Discord server can run /jambot-setup. This prevents unauthorized users from changing who can post setlists or approve songs.

How It Works

  1. Jam Leaders post setlist messages that JamBot detects and processes
  2. Approvers receive DM workflows to approve songs and create Spotify playlists
  3. When a jam leader posts a setlist:
    • JamBot detects and parses the message
    • Searches for each song on Spotify
    • Sends approval workflow to ALL configured approvers
    • Creates playlist when approved

LegacyEnvironment Variables

You can still use environment variables for initial configuration or as a fallback:

# .env file
DISCORD_JAM_LEADER_ID="123456789012345678"
DISCORD_ADMIN_ID="987654321098765432"
‚ö†Ô∏è Note: Database configuration takes precedence over environment variables. The bot will only use environment variables if no configuration exists in the database.

Database Storage

Configuration is stored in the SQLite database in the bot_configuration table:

CREATE TABLE bot_configuration (
    id INTEGER PRIMARY KEY,
    guild_id INTEGER UNIQUE NOT NULL,
    jam_leader_ids TEXT NOT NULL,      -- JSON array
    approver_ids TEXT NOT NULL,        -- JSON array
    updated_at TIMESTAMP NOT NULL,
    updated_by INTEGER NOT NULL
);

Migration from Environment Variables

If you're currently using environment variables:

  1. Keep your .env file as-is for bot token and Spotify credentials
  2. Run /jambot-setup in your Discord server to configure jam leaders and approvers
  3. The bot will now use the database configuration
  4. You can optionally remove DISCORD_JAM_LEADER_ID and DISCORD_ADMIN_ID from .env

Examples

Configure Multiple Jam Leaders

If you have 3 people who post setlists:

  1. Get each person's ID using /jambot-getid for each person
  2. Run /jambot-setup
  3. Enter the IDs separated by commas: 123456789, 234567890, 345678901

Configure Multiple Approvers

If you want 2 people to approve songs:

  1. Get their IDs using /jambot-getid
  2. Run /jambot-setup
  3. Enter the IDs separated by commas: 111222333, 444555666

Best Practices

  • Have at least 2 approvers for redundancy in case one is unavailable
  • Test the configuration by posting a test setlist after setup
  • Keep your .env file secure - it still contains sensitive Spotify credentials
  • Regularly review access - periodically check who has jam leader and approver permissions

Troubleshooting

"You need administrator permissions to use this command"

  • Make sure you have the Administrator role in your Discord server
  • Check your server role settings in Server Settings ‚Üí Roles
  • All configuration commands (/jambot-setup, /jambot-settings, /jambot-spotify-setup) require admin permissions

"The following user IDs are invalid..."

  • Verify the user IDs are correct (18-19 digit numbers)
  • Make sure the users are members of your Discord server
  • Use /jambot-getid @username to get the correct ID

"No approvers configured for this guild"

  • Run /jambot-setup to configure approvers
  • Make sure you entered at least one approver ID
  • Verify the configuration saved successfully

"Please run /jambot-setup first..."

  • You must run /jambot-setup before /jambot-settings
  • The bot needs essential configuration (jam leaders, approvers, Spotify credentials) before configuring advanced settings

"No Spotify credentials configured..."

  • Run /jambot-setup to add your Spotify app credentials (Client ID and Client Secret)
  • Make sure you created a Spotify Developer app first (see Step 1)
  • Verify you copied the Client ID and Client Secret correctly from your Spotify Developer Dashboard

Spotify authentication fails

  • Verify your Spotify app's Redirect URI matches your web server URL
  • Check that your Client ID and Client Secret are correct in /jambot-setup
  • Make sure your Spotify Developer app settings allow the redirect URI
  • Try deleting and recreating your Spotify app if issues persist

Configuration not taking effect

  • Try re-running the appropriate configuration command
  • Check the bot logs for any error messages in /app/logs/jambot.log
  • Verify the database file exists and is writable
  • Restart the bot if changes still don't take effect

Premium Features Setup

JamBot supports premium AI chord chart generation through an optional Premium API service. This feature enables AI-powered chord chart creation with a credit-based billing system.

üí° What is Premium? Premium features unlock AI-generated chord charts for any song. New servers get 5 free generations to try the feature, then can purchase credit packs to continue.

Premium Environment Variables

Add these variables to your .env file to enable premium features:

# Premium API Configuration
PREMIUM_API_BASE_URL=https://api.premium.jambot.app
PREMIUM_API_TIMEOUT=60

Variable Reference

  • PREMIUM_API_BASE_URL (string) - The base URL for the JamBot Premium API
    • Default: https://api.premium.jambot.app
    • Only change this for self-hosted premium API instances
  • PREMIUM_API_TIMEOUT (integer) - Request timeout in seconds
    • Default: 60
    • AI generation can take time; keep this value at 30+ seconds
    • Must be a positive integer

Enabling Premium for Your Server

Once environment variables are configured, server administrators can enable premium:

  1. Obtain an API token from the JamBot Premium Portal
  2. Run /jambot-premium-setup in your Discord server
  3. Enter your API token in the modal
  4. The bot will validate your token and display your credit balance

Troubleshooting Premium Configuration

"Premium API request timed out"

  • Increase PREMIUM_API_TIMEOUT to 90 or 120 seconds
  • Check your network connectivity to the premium API
  • Verify PREMIUM_API_BASE_URL is correct

"Invalid premium token"

  • Verify your token was copied correctly from the premium portal
  • Check that your token hasn't been revoked
  • Ensure the token is for the correct Discord server

"Connection refused to premium API"

  • Check that PREMIUM_API_BASE_URL is set correctly
  • Verify your server can reach external HTTPS endpoints
  • Check for firewall rules blocking outbound connections
